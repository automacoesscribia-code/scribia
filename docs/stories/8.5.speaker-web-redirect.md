## Story

**As a** speaker who accepted the invitation,
**I want** to be redirected to an informative page when accessing the web app,
**so that** I know I need to use the desktop app and can download it.

## Context

Speakers nao tem funcionalidade na web app — toda a interacao deles e via desktop app. Quando um speaker tenta acessar a web (apos set-password ou login direto), precisa ver uma pagina informativa em vez de um dashboard vazio.

## Acceptance Criteria

1. Middleware redireciona role=speaker para `/speaker` (nova rota)
2. Pagina `/speaker` mostra:
   - Logo SCRIBIA
   - Saudacao: "Ola, {userName}!"
   - Card informativo: "Use o app ScribIA Capture para gravar suas palestras"
   - Lista das palestras atribuidas ao speaker (titulo + evento + data)
   - Botao "Baixar ScribIA Capture" (link para download — placeholder por agora)
   - Botao "Sair" (logout)
3. Speaker nao pode acessar `/dashboard`, `/admin` ou `/portal` — redireciona para `/speaker`
4. Pagina styled com design system Scribia (dark theme, purple accents)
5. Layout responsivo: funciona em mobile (speaker pode acessar pelo celular)
6. Se speaker nao tem palestras: mensagem "Nenhuma palestra atribuida ainda. O organizador do evento entrara em contato."

// app/speaker/page.tsx (server component)
export default async function SpeakerPage() {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) redirect('/login')

  const { data: speaker } = await supabase
    .from('speakers')
    .select('id, name')
    .eq('user_id', user.id)
    .single()

  const { data: lectures } = await supabase
    .from('lectures')
    .select('title, scheduled_at, status, events(name)')
    .eq('speaker_id', speaker?.id)

  return (
    <div className="min-h-screen bg-bg flex items-center justify-center p-6">
      <div className="max-w-2xl w-full space-y-6">
        {/* Logo */}
        {/* Welcome card */}
        {/* Lectures list */}
        {/* Download button */}
        {/* Logout button */}
      </div>
    </div>
  )
}
