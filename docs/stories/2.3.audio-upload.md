## Story

**As a** operator,
**I want** the audio to be uploaded automatically in chunks during recording,
**so that** the audio is safely stored in the cloud even if the app crashes or the connection drops.

## Acceptance Criteria

1. Upload automático de chunks a cada 30 segundos durante gravação
2. Upload via Supabase Storage para path `audio-files/{event_id}/{lecture_id}/chunk_{index}.webm`
3. Retry automático com exponential backoff (max 3 tentativas) em caso de falha de rede
4. Indicador visual de progresso de upload: chunks enviados / total, bandwidth
5. Ao parar gravação, Tauri command `finalize_upload(lecture_id)` envia último chunk
6. Edge Function `finalize-audio` combina chunks em arquivo final `final.webm`
7. Metadata da palestra atualizada no DB: `audio_path`, `audio_duration_seconds`, `status → 'processing'`
8. Upload funciona em conexões instáveis (buffering local até reconectar)
9. Se app crashar, chunks já enviados estão seguros no Storage
10. Progress bar mostra: "Enviando chunk 5/12 (2.3 MB)" com porcentagem

## Dev Notes

**Upload flow:**
```
Recording thread → Chunks em temp dir
                      ↓
Upload manager thread → Supabase Storage (por chunk)
                      ↓
Stop recording → finalize_upload() → Edge Function
                      ↓
Edge Function → Combina chunks → Update DB → Queue transcription job
```

**Edge Function finalize-audio:**
```typescript
// supabase/functions/finalize-audio/index.ts
import { createClient } from '@supabase/supabase-js'
```

**Retry strategy:**
- Attempt 1: immediate
- Attempt 2: wait 1s
- Attempt 3: wait 4s
- After 3 fails: mark chunk as pending, retry when connection restored
