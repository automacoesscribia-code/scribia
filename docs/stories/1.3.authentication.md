## Story

**As a** user (organizador ou participante),
**I want** to register and login with email/password or magic link,
**so that** I can access the system with my correct role and permissions.

## Acceptance Criteria

1. Supabase Auth configurado com providers: email/password + magic link
2. Página `/login` com formulário de email + senha e opção "Entrar com link mágico"
3. Página `/register` com formulário: nome completo, email, senha, role (organizador/participante)
4. Ao registrar, `user_profiles` é criado automaticamente via database trigger
5. Middleware Next.js protege rotas: `/dashboard/*` (organizer only), `/portal/*` (participant only)
6. Redirect automático baseado em role: organizer → `/dashboard`, participant → `/portal`
7. Logout funcional que limpa sessão e redireciona para `/login`
8. Hook `useAuth()` que expõe: `user`, `profile`, `isLoading`, `signIn`, `signUp`, `signOut`
9. Sessão persistida via cookies (httpOnly) usando `@supabase/ssr`
10. Página de erro para acesso não autorizado (403)

## Dev Notes

**Auth flow:**
1. User registra → Supabase Auth cria `auth.users` → trigger cria `user_profiles`
2. Login → JWT com `user.id` → middleware verifica role → redirect
3. Cada request: middleware lê cookie → verifica JWT → injeta user context

**Supabase SSR setup:**
```typescript
// packages/supabase-client/src/server.ts
import { createServerClient as createClient } from '@supabase/ssr'
// Usa cookies do Next.js para persistir sessão
```

**Middleware pattern:**
```typescript
// apps/web/src/middleware.ts
// 1. getUser() via supabase server client
// 2. Se não autenticado → redirect /login
// 3. Se autenticado → check role em user_profiles
// 4. Organizer em /portal → redirect /dashboard
// 5. Participant em /dashboard → redirect /portal
```

**Database trigger:**
```sql
CREATE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.user_profiles (id, email, full_name, role)
  VALUES (new.id, new.email, COALESCE(new.raw_user_meta_data->>'full_name', ''), 'participant');
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```
