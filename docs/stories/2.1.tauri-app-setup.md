## Story

**As a** organizer/operator,
**I want** a desktop application that runs on Windows and macOS,
**so that** I can capture audio from lectures at events.

## Acceptance Criteria

1. Tauri 2 app compila e roda no Windows (WebView2) e macOS (WebKit)
2. Tela inicial com login via Supabase Auth (reutiliza `@scribia/supabase-client`)
3. Após login, mostra lista de eventos ativos do organizer (fetched via Supabase)
4. Seletor de evento e palestra para iniciar sessão de gravação
5. UI responsiva e limpa usando React + Tailwind (componentes de `@scribia/ui` quando aplicável)
6. Tauri config com app name "ScribIA Capture", ícone customizado, window size 800x600
7. Build gera instalador: `.msi` (Windows) e `.dmg` (macOS)
8. Auto-updater configurado via Tauri updater plugin (endpoint em GitHub Releases)
9. URI scheme `scribia://` registrado via Tauri deep-link plugin para deep links de outras apps (formato: `scribia://capture/{event_id}/{lecture_id}`)

## Dev Notes

**Tauri 2 dependencies (Cargo.toml):**
```toml
[dependencies]
tauri = { version = "2", features = ["tray-icon"] }
tauri-plugin-updater = "2"
tauri-plugin-shell = "2"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

**Frontend (Vite + React):**
- Usa `@tauri-apps/api` para bridge JS ↔ Rust
- Supabase client via `@scribia/supabase-client` (browser mode)
- Tailwind já configurado no Tauri template

**Window config (tauri.conf.json):**
```json
{
  "app": {
    "windows": [{
      "title": "ScribIA Capture",
      "width": 800,
      "height": 600,
      "resizable": true,
      "minWidth": 640,
      "minHeight": 480
    }]
  }
}
```

- Login funciona e persiste sessão
- Eventos e palestras carregam corretamente
