## Story

**As an** organizer,
**I want** to invite speakers by email with a real invitation that creates their account,
**so that** speakers receive an email, set their password, and have a working account linked to their lectures.

## Context

A pagina de speakers (Story 7.2) ja existe mas o "convite" apenas insere um registro na tabela `speakers` — nenhum email e enviado. Precisamos integrar com o sistema de convites real (`send-invitation` Edge Function) para que:

1. Organizador adiciona email do palestrante na pagina de speakers
2. Sistema cria registro em `speakers` + `invitations`
3. Email real e enviado via Supabase Auth `inviteUserByEmail()`
4. Link no email leva para `/auth/set-password`
5. Ao definir senha, conta e criada e vinculada ao registro speaker

**Importante:** Super admin tambem pode convidar speakers (mesmas permissoes que organizer para esta operacao).

## Acceptance Criteria

1. Botao "Enviar convite" na pagina de speakers chama Edge Function `send-invitation` com `role: 'speaker'`
2. Edge Function `send-invitation` atualizada para aceitar `role: 'speaker'` + campo `speaker_name`
3. Ao receber convite de speaker, Edge Function:
   a. Cria registro em `speakers` (name, email)
   b. Cria registro em `invitations` (email, role='speaker', speaker_id, event_id, invited_by)
   c. Chama `supabase.auth.admin.inviteUserByEmail()` com redirect para set-password
4. Email enviado ao palestrante contem link para definir senha
5. Pagina `/auth/set-password` funciona para role speaker (redireciona para pagina informativa apos definir senha)
6. Ao aceitar convite, trigger `handle_new_user()` vincula `speakers.user_id` automaticamente
7. Lista de pendentes na pagina de speakers mostra status real do convite (pending/accepted/expired)
8. Botao "Reenviar" faz re-invoke da Edge Function (novo email)
9. Super admin pode convidar speakers (mesma funcionalidade que organizer)
10. Organizador so pode convidar speakers para seus proprios eventos

## Dev Notes

**Edge Function update:**
```typescript
// Novo campo no body
interface InviteBody {
  email: string
  role: 'organizer' | 'participant' | 'speaker'
  event_id?: string
  speaker_name?: string  // novo
}

// Speaker-specific logic
if (role === 'speaker') {
  // 1. Criar speaker record
  const { data: speaker } = await supabaseAdmin
    .from('speakers')
    .insert({ name: speakerName || email.split('@')[0], email })
    .select('id')
    .single()

  // 2. Criar invitation com speaker_id
  const { data: invitation } = await supabaseAdmin
    .from('invitations')
    .insert({
      email, role: 'speaker', invited_by: userId,
      event_id, speaker_id: speaker.id
    })
    .select('id, token')
    .single()

  // 3. Enviar email
  await supabaseAdmin.auth.admin.inviteUserByEmail(email, {
    redirectTo: `${siteUrl}/auth/set-password?token=${invitation.token}`
  })
}
```

**Frontend call:**
```typescript
const response = await fetch(`${supabaseUrl}/functions/v1/send-invitation`, {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${session.access_token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email,
    role: 'speaker',
    event_id: activeEventId,
    speaker_name: name || undefined
  })
})
```
