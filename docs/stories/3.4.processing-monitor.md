## Story

**As a** organizer,
**I want** to see real-time processing status of each lecture,
**so that** I know when transcriptions and materials are ready.

## Acceptance Criteria

1. Componente `ProcessingStatus` mostra pipeline completa: Transcription → Summary → E-book → Playbook → Card
2. Cada step mostra: queued (cinza), processing (azul/spinner), completed (verde/check), failed (vermelho/x)
3. Progress bar geral da lecture (0-100%) baseado em processing_progress
4. Updates em tempo real via Supabase Realtime (subscription em `lectures` e `processing_jobs`)
5. Ao clicar em step failed, mostra error_message com opção "Reprocessar"
6. Botão "Reprocessar" cria novo job para o step falhado
7. Notificação quando processamento completa (todos os jobs done)
8. Visível tanto no Admin Panel (web) quanto no Desktop app

## Dev Notes

**Supabase Realtime subscription:**
```typescript
const channel = supabase
  .channel('processing')
  .on('postgres_changes',
    { event: 'UPDATE', schema: 'public', table: 'lectures', filter: `id=eq.${lectureId}` },
    (payload) => setProgress(payload.new.processing_progress)
  )
  .on('postgres_changes',
    { event: '*', schema: 'public', table: 'processing_jobs', filter: `lecture_id=eq.${lectureId}` },
    (payload) => updateJobStatus(payload)
  )
  .subscribe()
```
