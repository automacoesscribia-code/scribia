## Story

**As a** participant,
**I want** to listen to the lecture audio through a streaming player,
**so that** I can revisit the lecture content but without being able to download the audio file.

## Acceptance Criteria

1. Página `/portal/lectures/[id]` com player de áudio embutido
2. Player com controles: play/pause, seek bar, volume, velocidade (0.5x, 1x, 1.25x, 1.5x, 2x)
3. Áudio carregado via signed URL temporária (1h expiry) — sem URL permanente exposta
4. Sem botão de download no player
5. Right-click desabilitado sobre o player (oncontextmenu prevent)
6. Content-Disposition: inline nos headers do signed URL
7. Timer mostrando posição atual / duração total
8. Player funciona em mobile (touch controls)
9. Auto-mark `lecture_access.accessed_at` quando participante abre a lecture
10. Player mostra info: título, palestrante, data, tópicos (se disponíveis)

## Dev Notes

**Signed URL fetch:**
```typescript
// Via Edge Function (not direct Storage access)
const res = await fetch(`${SUPABASE_URL}/functions/v1/audio-stream/${lectureId}`, {
  headers: { Authorization: `Bearer ${session.access_token}` }
})
const { signed_url } = await res.json()
// signed_url expires in 1h, Content-Disposition: inline
```

**Anti-download measures:**
1. Signed URLs com expiração curta (1h)
2. Content-Disposition: inline (browser plays, doesn't download)
3. No download attribute on audio element
4. Context menu disabled
5. CSS hides browser's built-in download button

**Nota:** Estas medidas dificultam o download casual mas não impedem um usuário determinado (podem usar DevTools). Para proteção mais forte seria necessário DRM, mas isso está fora do escopo MVP.

**Track access:**
```typescript
// On component mount
await supabase.from('lecture_access')
  .update({ accessed_at: new Date().toISOString() })
  .eq('lecture_id', lectureId)
  .eq('user_id', user.id)
```
