## Story

**As a** system,
**I want** to generate summaries and segment lectures by topics using an AI provider (Gemini default),
**so that** participants can quickly understand the lecture content and navigate by topic.

## Acceptance Criteria

1. Edge Function `_jobs/generate-summary.ts` processa jobs tipo `summary`
2. Recebe transcript_text da lecture e envia para AI Provider (Gemini 2.5 Flash default, configurável via `AI_PROVIDER` env var)
3. AI gera: resumo executivo (300-500 palavras), lista de tópicos com timestamps aproximados
4. Resumo salvo em `lectures.summary`
5. Tópicos salvos em `lectures.topics` (text array)
6. Prompt otimizado para pt-BR com instruções específicas para conteúdo de palestras
7. Progress tracking: processing_progress atualizado (25%→50%)
8. Job status: queued → processing → completed/failed
9. Se falhar após 3 tentativas: marca failed, não bloqueia outros jobs downstream

## Dev Notes

**AI Provider call (model-agnostic via @scribia/shared):**
```typescript
import { createAIProvider } from '@scribia/shared/ai'

const ai = createAIProvider() // Gemini 2.5 Flash (default) ou Claude (via AI_PROVIDER env)

const result = await ai.generateStructured(
  `Analise a transcrição abaixo e gere:
1. Um resumo executivo (300-500 palavras) capturando os pontos principais
2. Uma lista de tópicos abordados na palestra

TRANSCRIÇÃO:
${transcript_text}`,
  { summary: 'string', topics: 'string[]' }
)
```

**Topics format:**
```
["Introdução e contexto", "Machine Learning na prática", "Casos de uso reais", "Q&A e conclusão"]
```
