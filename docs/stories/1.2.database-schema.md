## Acceptance Criteria

1. Supabase project local configurado (`supabase/config.toml`)
2. Migration inicial com todas as 7 tabelas: `user_profiles`, `events`, `speakers`, `lectures`, `event_participants`, `lecture_access`, `processing_jobs`
3. Todos os Ă­ndices criados: `idx_events_organizer`, `idx_events_status`, `idx_lectures_event`, `idx_lectures_speaker`, `idx_lectures_status`, `idx_event_participants_user`, `idx_processing_jobs_lecture`, `idx_processing_jobs_status`
4. RLS habilitado em todas as tabelas com policies conforme arquitetura:
   - `events`: organizers manage own, participants view active joined events
   - `lectures`: organizers manage by event, participants view with access
   - `user_profiles`: users manage own profile
   - `event_participants`: users view own, organizers manage by event
   - `lecture_access`: users view own, organizers manage by event
   - `processing_jobs`: organizers view by event's lectures
5. Seed data com 1 organizer, 1 evento, 2 speakers, 3 palestras, 2 participantes
6. TypeScript types gerados automaticamente via `supabase gen types typescript`
7. Types integrados no `@scribia/supabase-client` package
8. `supabase start` + `supabase db push` funcionam sem erros
