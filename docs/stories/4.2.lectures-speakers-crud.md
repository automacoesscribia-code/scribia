## Story

**As an** organizer,
**I want** to manage lectures and speakers within an event,
**so that** I can set up the complete program of the event before recording.

## Acceptance Criteria

1. Tab "Palestras" no evento: lista de lectures com título, palestrante, horário, status
2. Formulário de criação de lecture: título, descrição, palestrante (select/create), data/hora, duração estimada
3. Edição e exclusão de lectures (com confirmação)
4. Tab "Palestrantes" no evento: lista de speakers com nome, empresa, role
5. Formulário de criação de speaker: nome, email, bio, empresa, cargo, avatar
6. Ao criar lecture, pode selecionar speaker existente ou criar novo inline
7. Status da lecture visível: scheduled, recording, processing, completed, failed (com ícones)
8. Ordenação de lectures por data/hora agendada
9. Bulk action: selecionar múltiplas lectures para reprocessar
10. Link direto da lecture para a tela de gravação no desktop app (deep link)

## Dev Notes

**Supabase queries:**
```typescript
// Lectures with speaker info
const { data } = await supabase
  .from('lectures')
  .select('*, speaker:speakers(*)')
  .eq('event_id', eventId)
  .order('scheduled_at')

// Create lecture
const { data } = await supabase.from('lectures').insert({
  event_id: eventId,
  speaker_id: speakerId,
  title, description, scheduled_at, duration_minutes
}).select('*, speaker:speakers(*)').single()
```

**Deep linking (Tauri):**
```json
// tauri.conf.json
{ "plugins": { "deep-link": { "desktop": { "schemes": ["scribia"] } } } }
```
