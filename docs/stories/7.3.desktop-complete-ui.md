## Story

**As an** operator using the desktop app,
**I want** a professional recording interface with waveform, status monitoring, upload progress, and system log,
**so that** I can capture audio with confidence and visibility into the system state.

## Context

A referência visual (`referencias/scribia-desktop.html`) define uma interface profissional de gravação com:
- Window chrome (macOS traffic lights)
- Top bar com evento + status
- Timer com indicator REC
- Waveform canvas animado
- Device selector com ícone + specs
- Status pills (Servidor/Buffer/Upload)
- Controles circulares estilizados
- Painel lateral: metadados, upload progress, log do sistema

O código atual tem a estrutura básica mas usa CSS classes simples sem todos os elementos visuais.

**Design System:** Seguir `docs/design-system.md` — especificamente a seção 13 (Desktop App CSS).

## Acceptance Criteria

1. **Window Chrome:** traffic lights (red/yellow/green dots), título "ScribIA Desktop · v2.1.4" (font-mono), `-webkit-app-region: drag`
2. **Top bar:** Logo "SCRIBIA" (font-heading), info central (evento + sala/trilha), status dot pulsante "Conectado" (green)
3. **Timer display:**
   - Indicator badge "REC" com dot pulsante (quando gravando)
   - Indicator "PAUSADO" (yellow, quando pausado)
   - Timer: `00:42:17` (font-mono, 56px)
   - Subtexto contextual: "Pronto para gravar" / "Gravação em andamento" / "Pausado" / "Finalizada"
4. **Waveform card:**
   - Label "Forma de onda — entrada" + nível em dB (font-mono, purple-light)
   - **Canvas animado** com 80 barras, amplitudes variáveis via Math.sin + random, cor purple com alpha variável
   - Background: radial-gradient subtle no bottom
5. **Device selector:**
   - Ícone de microfone em box (purple-dim, border-purple)
   - Nome do dispositivo + specs (USB · 44.1kHz · Stereo)
   - Botão "Trocar" (ghost)
6. **Status pills (3, horizontal):**
   - Servidor: ícone check + "Conectado" (green)
   - Buffer: ícone battery + "Normal" (green)
   - Upload: ícone graph + percentual (yellow se < 100%)
7. **Controles (5 botões circulares):**
   - Settings (gear, 40px, secondary)
   - Pause (parallel bars, 52px, ghost)
   - Record (circle, 64px, purple glow — ou red quando gravando)
   - Stop (square, 52px, red tinted)
   - History (clock, 40px, secondary)
8. **Painel lateral direito (280px):**
   - Card "Metadados da sessão": evento, palestrante, título da palestra, data/hora, sala
   - Card "Upload em tempo real": label + percentual + barra (gradient purple→green) + detalhe "X MB enviados · Y MB pendentes · Z MB/s"
   - Card "Log do sistema": entries monospace com timestamp colorido (green=ok, blue=info, yellow=warn)
9. **Layout grid:** `grid-template-columns: 1fr 280px`, padding 24px, gap 20px
10. Animação fadeUp nos painéis ao carregar
11. Waveform canvas usa `requestAnimationFrame` para animação fluida
12. Status pills atualizam em tempo real via polling do status de captura

## Dev Notes

**Waveform canvas (da referência):**
```javascript
const bars = 80;
const barW = canvas.offsetWidth / bars;
function draw() {
  ctx.clearRect(0, 0, canvas.offsetWidth, 80);
  for (let i = 0; i < bars; i++) {
    const noise = Math.sin(i * 0.4 + t) * 0.5
                + Math.sin(i * 0.8 + t * 1.3) * 0.3
                + Math.random() * 0.2;
    const amp = (Math.abs(noise) * 28 + 4);
    const alpha = 0.4 + Math.abs(noise) * 0.6;
    ctx.fillStyle = `rgba(107,78,255,${alpha})`;
    ctx.beginPath();
    ctx.roundRect(x + 1, 40 - amp, barW - 2, amp * 2, 2);
    ctx.fill();
  }
  t += 0.05;
  requestAnimationFrame(draw);
}
```
