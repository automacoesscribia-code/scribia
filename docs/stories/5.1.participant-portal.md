## Story

**As a** participant,
**I want** to see my lectures with audio player, e-book previews, and waveform visuals,
**so that** I can consume the content from the events I attended.

## Context

A referência visual (`referencias/scribia-participant.html`) define uma experiência rica:
- Top nav com logo, pill de evento, avatar do usuário
- Hero section com saudação, stats
- Player destacado com waveform, controles, preview de e-book inline
- Grid de lecture cards com mini-waveforms animados, chips de status, botões de ação

Atualmente `/portal` é apenas um stub vazio.

**Design System:** Seguir rigorosamente `docs/design-system.md`.

## Acceptance Criteria

1. **Top nav (sticky):** Logo "SCRIBIA" (esquerda), pill de evento (centro), avatar + nome (direita)
2. **Hero section:**
   - Saudação: "Bem-vindo de volta,"
   - Título: "Minhas Palestras" (font-heading 28px, extrabold)
   - Subtítulo: "Acesse os conteúdos das palestras que você assistiu"
   - Stats row: X palestras assistidas | Y e-books disponíveis | Z participantes no evento
3. **Player section (destacado):**
   - Label "● Em reprodução" (purple-light)
   - Título da palestra (font-heading 18px, extrabold) + palestrante + duração
   - Badge "Download não disponível" (red tinted, com ícone)
   - **Waveform visual:** 120 barras verticais, played (purple) vs unplayed (bg4), alturas variáveis
   - **Controles:** tempo atual | progress bar com handle | tempo total | botões prev/play/next
   - Botão play: 44px, circle, purple com glow
   - **E-book preview:** card bg3, título "E-book gerado · Capítulo 1", conteúdo markdown (h4 + p + ul), link "Ler completo →", fade-out gradient
4. **Grid de palestras (3 colunas):**
   - Section header: "Todas as minhas palestras" + link "Filtrar"
   - **Lecture card structure:**
     - Top area com gradient background (cores variadas por palestra) + mini-waveform animado (7 barras)
     - Body: título (font-heading 14px bold), palestrante, chip de status + duração
     - Footer: botões "Ouvir" (primary) + "E-book" (ghost) — ou "Em breve" (disabled) se processando
   - Animação fadeUp com stagger
5. **Status chips:** Transcrito (green), E-book Pronto (purple), Processando (yellow)
6. Dados via RLS: só palestras com `lecture_access` para o user logado
7. Se não houver palestras: empty state "Nenhuma palestra disponível ainda"
8. Layout responsivo: 1 col mobile, 2 cols tablet, 3 cols desktop
9. Rota `/portal` com layout próprio (sem sidebar do dashboard)
10. Streaming de áudio via signed URL do Supabase Storage (1h expiry, sem download)

## Dev Notes

**Supabase queries:**
```typescript
// Palestras do participante
const { data: lectures } = await supabase
  .from('lecture_access')
  .select(`
    lecture:lectures(
      id, title, status, duration_seconds,
      speaker:speakers(name),
      event:events(name),
      ebook_content, summary
    )
  `)
  .eq('user_id', user.id)

// Signed URL para streaming
const { data: signedUrl } = await supabase.storage
  .from('audio')
  .createSignedUrl(`${eventId}/${lectureId}/combined.webm`, 3600)
```

**Mini-waveform CSS animation:**
```css
.mini-wave span {
  width: 3px;
  border-radius: 2px;
  background: var(--purple-light);
  animation: mw var(--d, 0.8s) ease-in-out infinite alternate;
  opacity: 0.6;
}
@keyframes mw {
  from { height: 6px; }
  to { height: var(--h, 28px); }
}
```
