## Story

**As an** organizer,
**I want** to create, view, edit, and delete events in the admin panel,
**so that** I can manage my events and their details.

## Acceptance Criteria

1. Página `/dashboard` com lista de eventos do organizer (cards com nome, data, status, qtd palestras)
2. Página `/dashboard/events/new` com formulário: nome, descrição, data início/fim, local, imagem de capa
3. Página `/dashboard/events/[id]` com detalhes do evento e tabs: Palestras, Palestrantes, Participantes, Analytics
4. Edição inline dos campos do evento (click to edit)
5. Botão de excluir evento com confirmação (soft delete → status 'archived')
6. Filtros: por status (draft, active, completed, archived), busca por nome
7. Botão "Ativar Evento" muda status de draft → active
8. Validação: nome obrigatório, data fim >= data início
9. Upload de imagem de capa via Supabase Storage
10. Todas as operações respeitam RLS (organizer só vê seus eventos)

## Dev Notes

**Form validation (Zod):**
```typescript
const eventSchema = z.object({
  name: z.string().min(1, 'Nome é obrigatório'),
  description: z.string().optional(),
  start_date: z.string().date(),
  end_date: z.string().date(),
  location: z.string().optional(),
}).refine(d => d.end_date >= d.start_date, {
  message: 'Data fim deve ser >= data início',
  path: ['end_date']
})
```

**Supabase queries (RLS enforced):**
```typescript
// List events (RLS filters by organizer_id = auth.uid())
const { data } = await supabase
  .from('events')
  .select('*, lectures(count)')
  .order('start_date', { ascending: false })

// Create event
const { data } = await supabase.from('events').insert({
  name, description, start_date, end_date, location, organizer_id: user.id
}).select().single()
```
