## Story

**As an** organizer,
**I want** a rich dashboard with stats, recent lectures, quick actions, and processing progress,
**so that** I have a complete overview of my event's status at a glance.

## Context

A página `/dashboard` atual mostra apenas uma lista de eventos com filtros. A referência visual (`referencias/scribia-dashboard.html`) define um dashboard completo com:
- 4 stat cards com métricas do evento
- Tabela de palestras recentes com status e ações
- Grid de ações rápidas
- Barras de progresso de processamento

**Design System:** Seguir rigorosamente `docs/design-system.md` — tokens Scribia, font-heading (Syne), dark theme.

## Acceptance Criteria

1. Página `/dashboard` mostra dashboard do evento ativo (ou mais recente) do organizador
2. **Stats Grid (4 cards):** Palestras (total/planejadas), Áudio convertido (horas transcritas), E-books gerados (total/pendentes), Downloads (total + taxa de engajamento)
3. Cada stat card tem: label, valor grande (font-heading 32px), subtítulo, badge colorido, barra de acento colorida no topo (purple/green/yellow/teal)
4. **Tabela "Palestras recentes":** 5 últimas palestras com colunas: Palestra (título), Palestrante (avatar initials + nome), Status (Chip: Transcrito/E-book Pronto/Processando), Ações (ícones: visualizar, download)
5. Card action "Ver todas →" navega para detalhes do evento
6. **Ações rápidas (2x2 grid):** Add Palestrante, Add Participante, Gerar relatório, Analytics — cada com ícone em box roxo + label + sublabel
7. **Processamento de áudio:** 4 barras de progresso — Transcrição (green), E-books (purple), Cards de divulgação (yellow), Playbooks (teal) — cada com label + valor/max
8. **Topbar:** título "Dashboard" (font-heading), subtítulo com nome do evento + datas, botões "Exportar relatório" (ghost) e "Nova palestra" (primary purple com glow)
9. Animação fadeUp com stagger nos stat cards
10. Se não houver evento ativo, mostrar empty state com CTA "Criar primeiro evento"
11. Dados vêm de queries Supabase com RLS (organizador só vê seus dados)

## Dev Notes

**Supabase queries:**
```typescript
// Evento ativo mais recente
const { data: event } = await supabase
  .from('events')
  .select('*')
  .eq('status', 'active')
  .order('start_date', { ascending: false })
  .limit(1)
  .single()

// Stats do evento
const { count: lectureCount } = await supabase
  .from('lectures')
  .select('*', { count: 'exact', head: true })
  .eq('event_id', event.id)

// Palestras recentes com speaker
const { data: recentLectures } = await supabase
  .from('lectures')
  .select('id, title, status, speakers(name)')
  .eq('event_id', event.id)
  .order('scheduled_at', { ascending: false })
  .limit(5)
```
