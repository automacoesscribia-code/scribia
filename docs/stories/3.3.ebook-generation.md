## Story

**As a** system,
**I want** to automatically generate e-books and playbooks from lecture transcriptions,
**so that** participants receive structured, readable content derived from the lecture.

## Acceptance Criteria

1. Edge Function `_jobs/generate-ebook.ts` gera e-book a partir da transcrição + resumo
2. Edge Function `_jobs/generate-playbook.ts` gera playbook (guia prático) a partir da transcrição + tópicos
3. E-book: documento estruturado com introdução, capítulos por tópico, conclusão (formato Markdown)
4. Playbook: guia prático com checklists, action items, key takeaways (formato Markdown)
5. Conteúdo gerado via AI Provider (Gemini 2.5 Flash default, configurável via `AI_PROVIDER`) com prompts otimizados para cada formato
6. Markdown convertido para PDF via `pdf-lib` (Deno-native, sem dependências externas) usando `ScribiaPdfBuilder` de `@scribia/shared`
7. PDFs salvos no Supabase Storage: `materials/{event_id}/{lecture_id}/ebook.pdf` e `playbook.pdf`
8. URLs salvas em `lectures.ebook_url` e `lectures.playbook_url`
9. Progress tracking: 50%→75% (ebook), 75%→90% (playbook)
10. Jobs rodam em paralelo (não dependem um do outro)

## Dev Notes

**E-book prompt (Claude):**
```
Gere um e-book em Markdown a partir desta palestra. Estruture com:
- Capa: título, palestrante, evento
- Introdução: contexto e importância do tema
- Capítulos: um por tópico, expandindo o conteúdo da transcrição
- Conclusão: síntese e próximos passos
- Referências (se mencionadas na palestra)

Tom: profissional, acessível, em português brasileiro.
Tamanho: 3000-5000 palavras.
```

**Playbook prompt (Claude):**
```
Gere um playbook prático em Markdown a partir desta palestra. Estruture com:
- Objetivo: o que o leitor vai aprender
- Key Takeaways: 5-7 pontos principais
- Para cada tópico:
  - Resumo do conceito
  - Checklist de ações práticas
  - Dicas e melhores práticas
- Plano de ação: próximos passos concretos

Tom: direto, prático, actionable. Português brasileiro.
```

**PDF generation via pdf-lib (Deno-native — ADR-001 v2):**
```typescript
import { ScribiaPdfBuilder } from '@scribia/shared/pdf'

const builder = new ScribiaPdfBuilder()
await builder.addCover(lecture.title, speaker.name, scribiaLogo)
await builder.addTableOfContents(chapters.map(c => c.title))
for (const chapter of chapters) {
  await builder.addChapter(chapter.title, chapter.content)
}
await builder.addFooter()
const pdfBytes = await builder.build()
// Upload para Supabase Storage
```

**Nota arquitetural:** `pdf-lib` roda nativamente em Deno sem dependências externas (sem puppeteer, sem browser). `ScribiaPdfBuilder` é definido em Story 1.5 e reutilizado por Stories 4.5 e 6.2.
