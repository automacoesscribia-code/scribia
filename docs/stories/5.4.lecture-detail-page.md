# Story 5.4: Lecture Detail Page (Unified)

## Status
Draft

executor: "@dev"
quality_gate: "@architect"
quality_gate_tools: [typecheck, vitest, lint]

## Story

**As a** participant,
**I want** a unified lecture page with all content (áudio, resumo, e-book, playbook),
**so that** I can access everything about a lecture in one place.

## Acceptance Criteria

1. Página `/portal/lectures/[id]` como hub central da palestra
2. Header: título, palestrante (nome + foto + bio), evento, data
3. Seção "Resumo": resumo da palestra (lectures.summary) com formatação
4. Seção "Tópicos": lista de tópicos abordados
5. Seção "Áudio": player integrado (componente de Story 5.2)
6. Seção "Materiais": cards para E-book e Playbook com botões "Ler" e "Download PDF"
7. Se conteúdo em processamento: mostrar ProcessingStatus (componente de Story 3.4)
8. Transcrição completa acessível via botão "Ver transcrição completa" (expandível ou modal)
9. Breadcrumb: Portal > Evento > Palestra
10. SEO meta tags para compartilhamento (og:title, og:description)

**Data fetch (Server Component):**

const { data: lecture } = await supabase
  .from('lectures')
  .select('*, speaker:speakers(*), event:events(name)')
  .eq('id', params.id)
  .single()
