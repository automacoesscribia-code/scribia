## Story

**As an** organizer,
**I want** to edit or replace the audio file of a lecture,
**so that** I can fix audio issues or use a better recording.

## Acceptance Criteria

1. Na página de detalhe da lecture (admin): seção "Áudio" com player + opções
2. Botão "Substituir Áudio": upload de novo arquivo de áudio (mp3, wav, webm, m4a)
3. Ao substituir: deleta áudio anterior do Storage, faz upload do novo
4. Ao substituir: reseta status da lecture para 'processing' e cria novos processing_jobs
5. Confirmação antes de substituir: "Isso irá reprocessar toda a palestra. Continuar?"
6. Botão "Editar Áudio": funcionalidade básica de trim (cortar início/fim) via Web Audio API nativa (zero dependências externas)
7. Trim: selecionar start/end time visualmente (wavesurfer.js) → gerar novo arquivo cortado via Web Audio API (`AudioBuffer.getChannelData()` + slice temporal)
8. Histórico: manter referência ao áudio original (backup)
9. Validação: max file size 500MB, formatos aceitos
10. Progress bar durante upload de novo arquivo

## Dev Notes

**Audio trim no browser (Web Audio API):**
```typescript
// Alternativa: usar ffmpeg-wasm no browser
import { FFmpeg } from '@ffmpeg/ffmpeg'
const ffmpeg = new FFmpeg()
await ffmpeg.exec(['-i', 'input.webm', '-ss', startTime, '-to', endTime, '-c', 'copy', 'output.webm'])
```

**Waveform visualization:**
```typescript
import WaveSurfer from 'wavesurfer.js'
import RegionsPlugin from 'wavesurfer.js/dist/plugins/regions'
// Permite seleção visual de região para trim
```

**Backup pattern:**
- Original: `audio-files/{event_id}/{lecture_id}/final.webm`
- Backup: `audio-files/{event_id}/{lecture_id}/backup_{timestamp}.webm`
- Novo: `audio-files/{event_id}/{lecture_id}/final.webm` (overwrite)
