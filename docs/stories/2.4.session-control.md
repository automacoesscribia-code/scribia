## Story

**As a** operator,
**I want** a complete session control interface for managing lecture recordings,
**so that** I can easily start, monitor, pause, and finish recordings with full visibility of the process.

## Acceptance Criteria

1. Tela de "Sessão de Gravação" com: info da palestra (título, palestrante, evento), timer, VU meter, upload status
2. Fluxo completo: Selecionar dispositivo → Iniciar gravação → Monitorar → Parar → Confirmar finalização
3. Botão "Iniciar" muda lecture status para `recording` no DB (via Supabase)
4. Botão "Parar" mostra diálogo de confirmação antes de finalizar
5. Após finalizar, mostra resumo: duração total, chunks enviados, status de processamento
6. Metadados registrados no DB: início, fim, duração, device usado
7. Lista de gravações recentes no menu lateral com status (recording, processing, completed, failed)
8. Possibilidade de retomar upload de chunks pendentes (se app foi fechado durante gravação)

## Dev Notes

**Session flow:**
```
Select Event → Select Lecture → Select Device → Start
     ↓
Recording Screen: Timer | VU Meter | Upload Progress
     ↓
Stop → Confirmation Dialog → Finalize → Summary Screen
```

**Status transitions:**
- `scheduled` → `recording` (on start)
- `recording` → `processing` (on finalize-audio Edge Function)
- `processing` → `completed` (after all jobs done)
- Any → `failed` (on unrecoverable error)

### Testing
- Fluxo completo: start → record 30s → stop → verify DB updated
- Verificar que lecture.status muda corretamente
- Testar recovery: fechar app durante gravação, reabrir, verificar opção de resume
- Verificar lista de gravações recentes atualiza em tempo real
