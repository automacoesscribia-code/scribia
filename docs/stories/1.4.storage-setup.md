## Story

**As a** system,
**I want** Supabase Storage buckets configured for audio files and generated materials,
**so that** audio can be uploaded securely and materials can be downloaded by authorized users.

## Acceptance Criteria

1. Bucket `audio-files` criado: private, max file size 500MB, allowed MIME types: `audio/*`
2. Bucket `materials` criado: private, max file size 50MB, allowed MIME types: `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `image/*`
3. Storage policies no bucket `audio-files`:
   - INSERT: authenticated users (upload)
   - SELECT: via signed URLs only (sem acesso direto)
4. Storage policies no bucket `materials`:
   - INSERT: service_role only (edge functions geram)
   - SELECT: users com `lecture_access` para a lecture correspondente
5. Utility function `getSignedAudioUrl(lectureId)` que retorna URL temporária (1h) com `Content-Disposition: inline`
6. Utility function `getSignedMaterialUrl(lectureId, type)` para download de materiais
7. Upload path pattern: `audio-files/{event_id}/{lecture_id}/chunk_{index}.webm` e `audio-files/{event_id}/{lecture_id}/final.webm`
8. Materials path pattern: `materials/{event_id}/{lecture_id}/{type}.{ext}`

## Dev Notes

**Signed URL para streaming (sem download):**
```typescript
const { data } = await supabase.storage
  .from('audio-files')
  .createSignedUrl(path, 3600, {
    download: false, // Content-Disposition: inline
  })
```

**Storage policy para materials com RLS check:**
```sql
CREATE POLICY "users_download_accessible_materials"
ON storage.objects FOR SELECT
USING (
  bucket_id = 'materials' AND
  -- Extract lecture_id from path: materials/{event_id}/{lecture_id}/...
  EXISTS (
    SELECT 1 FROM public.lecture_access la
    WHERE la.user_id = auth.uid()
    AND la.lecture_id::text = (storage.foldername(name))[2]
  )
);
```
