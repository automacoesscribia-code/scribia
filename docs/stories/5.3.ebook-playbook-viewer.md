## Story

**As a** participant,
**I want** to read e-books and playbooks generated from lectures,
**so that** I can study the structured content at my own pace.

## Acceptance Criteria

1. Página `/portal/lectures/[id]/ebook` com visualização do e-book
2. Página `/portal/lectures/[id]/playbook` com visualização do playbook
3. Renderização de Markdown para HTML com formatação rica (headings, lists, code blocks, quotes)
4. Navegação por tópicos/capítulos (sidebar ou top nav)
5. Botão "Download PDF" para baixar versão PDF do e-book/playbook
6. Responsivo e legível em mobile
7. Track de download: incrementa `lecture_access.download_count` no download
8. Loading state enquanto conteúdo é buscado
9. Se e-book/playbook ainda não gerado: mostra mensagem "Conteúdo em processamento..."

## Dev Notes

**Markdown rendering:**
```typescript
import ReactMarkdown from 'react-markdown'
import remarkGfm from 'remark-gfm'

<ReactMarkdown remarkPlugins={[remarkGfm]}>
  {markdownContent}
</ReactMarkdown>
```

**Table of Contents extraction:**
```typescript
// Extrair headings do Markdown para sidebar
const headings = markdownContent.match(/^#{1,3} .+$/gm)
```

**Download tracking:**
```typescript
const handleDownload = async () => {
  // 1. Get signed URL
  const { signed_url } = await getSignedMaterialUrl(lectureId, 'ebook')
  // 2. Increment counter
  await supabase.rpc('increment_download_count', { p_lecture_id: lectureId, p_user_id: user.id })
  // 3. Trigger download
  window.open(signed_url, '_blank')
}
```
