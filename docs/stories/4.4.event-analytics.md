## Story

**As an** organizer,
**I want** to see analytics and metrics about my event,
**so that** I can understand engagement and measure the event's success.

## Acceptance Criteria

1. Tab "Analytics" no evento com dashboard de métricas
2. KPIs principais: total palestras, palestras processadas, áudios convertidos, e-books gerados, taxa de downloads
3. Gráfico de engajamento: participantes ativos por palestra (quem acessou)
4. Tabela: por palestra — views, downloads de ebook, downloads de playbook, plays do áudio
5. Métrica de taxa de conversão: participantes registrados vs que acessaram conteúdo
6. Edge Function `event-analytics` que agrega dados
7. Botão "Gerar Relatório Final" que navega para funcionalidade da Story 4.5 (desabilitado se evento não está 'completed')
8. Dados atualizados em tempo real (ou refresh manual com botão)

## Dev Notes

**Analytics aggregation query:**
```sql
SELECT
  e.id as event_id,
  COUNT(DISTINCT l.id) as total_lectures,
  COUNT(DISTINCT l.id) FILTER (WHERE l.status = 'completed') as processed_lectures,
  COUNT(DISTINCT l.id) FILTER (WHERE l.audio_path IS NOT NULL) as audios_converted,
  COUNT(DISTINCT l.id) FILTER (WHERE l.ebook_url IS NOT NULL) as ebooks_generated,
  COALESCE(SUM(la.download_count), 0) as total_downloads,
  COUNT(DISTINCT la.user_id) FILTER (WHERE la.accessed_at IS NOT NULL) as active_participants,
  COUNT(DISTINCT ep.user_id) as total_participants
FROM events e
LEFT JOIN lectures l ON l.event_id = e.id
LEFT JOIN lecture_access la ON la.lecture_id = l.id
LEFT JOIN event_participants ep ON ep.event_id = e.id
WHERE e.id = $1
GROUP BY e.id;
```
