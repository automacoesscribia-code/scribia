## Story

**As an** organizer,
**I want** to manage event participants and control their access to lectures,
**so that** only registered participants can access event content.

## Acceptance Criteria

1. Tab "Participantes" no evento: lista com nome, email, data de registro, status (attended/not)
2. Adicionar participante: por email (convite) ou importar CSV (nome, email)
3. Ao adicionar participante: cria user_profiles se não existe, cria event_participant, cria lecture_access para todas as lectures do evento
4. Remover participante: remove event_participant e lecture_access (não deleta user)
5. Marcar presença: checkbox "Presente" por participante
6. Filtro: busca por nome/email, filtro por presença
7. Contagem total: "X participantes registrados, Y presentes"
8. Exportar lista de participantes como CSV
9. Enviar convite por email (via Supabase Auth invite ou magic link)
10. Quando nova lecture é adicionada ao evento, lecture_access é criada automaticamente para todos os participantes existentes

## Dev Notes

**Database trigger para auto lecture_access:**
```sql
CREATE FUNCTION public.auto_grant_lecture_access()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.lecture_access (lecture_id, user_id)
  SELECT NEW.id, ep.user_id
  FROM public.event_participants ep
  WHERE ep.event_id = NEW.event_id
  ON CONFLICT DO NOTHING;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_lecture_created
  AFTER INSERT ON public.lectures
  FOR EACH ROW EXECUTE FUNCTION public.auto_grant_lecture_access();
```
